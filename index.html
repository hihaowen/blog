图书馆借书分析

Posted on 2019-02-20 | In 程序设计

用面向对象思想来分析的思路

具体步骤:

确定场景

  • 图书入库 (StoreBo)
  • 借书 (BorrowBo)
  • 还书 (ReturnBo)

确定参与者

  • 图书
  • 人
  • 图书馆

确定参与者中具体哪些是对象,哪些是服务

  • 图书 =》 对象
  • 人 =〉 对象
  • 图书馆 =》 服务

4、确定参与者对象的基本特征

  • 图书: 书名、作者、发行时间、字数
  • 人: 用户ID
  • 图书馆: 图书入库、借书、还书、图书查询

5、确定参与者会涉及到的角色

  • 图书 => 无
  • 人 => 借书人
  • 图书馆 => 无

6、确定参与者对象在经历活动之后发生的变化

  • 图书 => 无
  • 人 => 多了几本书
  • 图书馆 => 多了或者少了几本书

7、确定参与对象间的中间产物

  • 图书和图书馆之间 => 图书馆某图书信息
  • 人和图书之间 => 某人借的书的信息
  • 人和图书馆之间 => 人的借书记录信息

读书思路

Posted on 2017-08-21

探索(整个时间的 5% ~ 10%)

1. 观点是什么？
2. 方法是什么？
3. 结论是什么？
4. 标记重点

理解(整个时间的70%~80%)

1. 什么论据来支撑观点、方法、结论？
2. 这些观点、论证和结论的缺陷是什么？
3. 你怎么认为这些观点、证据和结论？
4. 作者怎样（或者怎么能）回应这些缺陷，和你的批评？

回忆和记笔记

1. 想象你正在和作者进行一个对话，或者是一个普通的辩论
2. 想象给书一个考试。考试的题目是什么，你是怎样回答的？
    → 记录重要元素，更新到自己理解框架
    → 平均每 100 页输出 1 - 3 页笔记
    → 笔记可以快速的定位到书中内容
